Content Management

How to add, update, and remove content from the offline library

Config Files

All content is defined in plain text config files on the Pi at /srv/offline/scripts/config/. Edit these files to add or remove content, then trigger a sync.

FileControlsUpdate detection
zim-list.conf Wikipedia, Wiktionary, and other ZIM encyclopedias Auto — checks remote filename (date-versioned monthly)
pdf-sources.conf PDF guides and manuals Manual — local SHA-256 only; delete file to force re-download
book-list.conf EPUBs from Project Gutenberg and Standard Ebooks Manual — filename existence; delete file to re-fetch
video-list.conf YouTube videos, playlists, channels; archive.org clips Manual — yt-dlp archive ID; remove ID from archive to re-fetch
map-regions.conf OSM map regions (PBF → MBTiles) and USGS topos Auto — compares Geofabrik MD5 checksum

Triggering a Sync

Full sync (all modules)

sudo systemctl start survive-sync.service
journalctl -u survive-sync -f          # follow log; Ctrl+C to detach

Specific modules only

SYNC_MODULES='pdfs' sudo systemctl start survive-sync.service
SYNC_MODULES='pdfs books' sudo systemctl start survive-sync.service
# module names: zim  pdfs  books  maps  video

Check last sync result

journalctl -u survive-sync -n 100 --no-pager
ls /srv/offline/logs/

Adding a PDF

Edit /srv/offline/scripts/config/pdf-sources.conf and add a line in this format (fields separated by a single tab):

url<TAB>local_filename<TAB>category_dir<TAB>priority<TAB>description

Example:

https://example.gov/guide.pdf	01-medical__example__field-guide__2023.pdf	01-medical	P1	Example Field Guide 2023

Fields

FieldNotes
urlDirect link to the PDF. Must be publicly accessible.
local_filename Use the convention: category__source__title__year.pdf
Lowercase, hyphens for spaces, double-underscore as separator.
category_dir Subdirectory under /srv/offline/pdfs/ — see categories below.
priority P1 Lifesaving / essential   P2 High value   P3 Educational
descriptionHuman-readable label for logs and future reference.

Categories

DirectoryContents
00-start-herePreparedness checklists and family plans
01-medicalFirst aid, trauma care, disease, health
02-waterPurification, sanitation, water systems
03-foodCanning, preservation, storage, cooking
04-agricultureFarming, gardening, livestock, soil
05-shelterConstruction, fortification, hazard mitigation
06-powerSolar, generators, batteries, propane
07-repairTools, vehicles, plumbing, electrical, survival
08-commsHam radio, emergency communications, ICS
09-navigationMaps, topo, land nav, tides
10-securityNIMS, ICS, community preparedness
11-referenceLegal, policy, frameworks
12-technologyHardware docs, server references

To force a re-download

The PDF sync only re-downloads if the local file is missing or its hash doesn't match the stored hash. To force a fresh copy:

rm /srv/offline/pdfs/01-medical/01-medical__who__basic-emergency-care__2018.pdf
# Then run the sync — it will re-download that file only
SYNC_MODULES='pdfs' sudo systemctl start survive-sync.service

Adding a Video

Edit /srv/offline/scripts/config/video-list.conf and add a line (tab-separated):

type<TAB>id_or_url<TAB>category<TAB>subcategory<TAB>title

Types

Typeid_or_urlLimit
video11-char YouTube video ID, or a full URL (including archive.org)1 video
playlistYouTube playlist URL20 most recent
channelYouTube channel URL10 most recent

Examples

# Single YouTube video (ID only)
video	M4ACYp75mjU	first-aid	cpr	AHA Hands-Only CPR

# archive.org direct MP4 URL
video	https://archive.org/download/BigBuckBunny_328/BigBuckBunny_512kb.mp4	morale	family	Big Buck Bunny

# YouTube channel (top 10 videos)
channel	https://www.youtube.com/@StoptheBleed	first-aid	bleeding	Stop the Bleed channel

Categories

Use any of: first-aid   repair   power   food   morale — or add a new category name; the directory will be created automatically.

To re-download a video

yt-dlp records every downloaded video ID in /srv/offline/video/.yt-dlp-archive.txt. To re-fetch a specific video, remove its line from that file:

# Find the entry (format: "youtube <ID>")
grep M4ACYp75mjU /srv/offline/video/.yt-dlp-archive.txt

# Remove it
sed -i '/M4ACYp75mjU/d' /srv/offline/video/.yt-dlp-archive.txt

# Re-run video sync
SYNC_MODULES='video' sudo systemctl start survive-sync.service

Adding a ZIM (Wikipedia / Encyclopedia)

Edit /srv/offline/scripts/config/zim-list.conf and add a line (pipe-separated):

slug | kiwix_subdir | description

Example:

stackoverflow_en_all     | stackoverflow   | Stack Overflow Q&A (programming)
gutenberg_en_all         | gutenberg       | Project Gutenberg full library

Finding slugs

Browse available ZIM files at library.kiwix.org or the download index at download.kiwix.org/zim/. The slug is the filename prefix before the date stamp (e.g. wikipedia_en_all_maxi from wikipedia_en_all_maxi_2024-11.zim). The kiwix_subdir is the subdirectory on the download server (e.g. wikipedia, wiktionary, stackoverflow).

ZIM files update automatically — the sync fetches the newest dated version and removes the old one. No manual re-download needed.

Adding a Book

Edit /srv/offline/scripts/config/book-list.conf and add a line (tab-separated):

source<TAB>id_or_slug<TAB>local_filename_base<TAB>category<TAB>priority<TAB>title

Sources

Sourceid_or_slugExample
gutenberg Numeric book ID from gutenberg.org/ebooks/ gutenberg 1342 pride-and-prejudice--austen classics P3 Pride and Prejudice
standardebooks Path from standardebooks.org/ebooks/ — e.g. author-name/book-title standardebooks jack-london/the-call-of-the-wild call-of-the-wild--london morale P3 The Call of the Wild

Categories

Use any of: practical   reference   morale   family   classics — or any name you like.

To re-download a book

rm /srv/offline/books/epub/robinson-crusoe--defoe.epub
SYNC_MODULES='books' sudo systemctl start survive-sync.service

Adding a Map Region

Edit /srv/offline/scripts/config/map-regions.conf and add a line (tab-separated):

geofabrik<TAB>region/state-identifier<TAB>state-slug<TAB>north-america/us<TAB>description

Example — adding New Jersey:

geofabrik	north-america/us/new-jersey	new-jersey	north-america/us	New Jersey OSM street map

The region/state-identifier is the Geofabrik path used to build the download URL. Browse regions at download.geofabrik.de.

Maps update automatically — the sync checks the Geofabrik MD5 checksum on each run and only re-downloads and re-converts if the source data has changed.

Removing Content

Comment out or delete the line from the appropriate config file, then delete the file(s) manually:

# Remove a PDF
rm /srv/offline/pdfs/04-agriculture/04-agriculture__misc__peasant-farming-techniques__2005.pdf

# Remove a ZIM (comment out its line in zim-list.conf first, then delete)
rm /srv/offline/kiwix/zim/wikipedia_en_simple_all_maxi_2024-11.zim

# Remove a book
rm /srv/offline/books/epub/moby-dick--melville.epub
# Also remove from Calibre
calibredb remove --with-library /srv/offline/books/calibre-library <ID>

If you only delete the file without removing the line from the config, the sync will re-download it on the next run.

Deploying Config Changes

Config files live in the git repo. The recommended workflow:

  1. Edit the config file on your laptop
  2. git push
  3. On the Pi: cd ~/survive-sync && git pull && sudo bash install.sh
  4. The next weekly sync (or a manual trigger) picks up the new entries

Or edit directly on the Pi if you need a quick change without going through git:

ssh joe@survive.travel
nano /srv/offline/scripts/config/pdf-sources.conf
SYNC_MODULES='pdfs' sudo systemctl start survive-sync.service
Direct edits on the Pi will be overwritten the next time install.sh runs. Commit changes back to the repo to make them permanent.
