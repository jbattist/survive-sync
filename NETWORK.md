# Battistello Network — WireGuard Split Tunnel

**Last Updated:** March 2026

---

## Overview

| Role | Device | LAN Subnet | Tunnel IP |
|---|---|---|---|
| VPN Server | UniFi Dream Router 7 (home) | 192.168.1.0/24 | 10.10.10.10 |
| VPN Client | GL.iNet Beryl AX MT3000 (travel) | 192.168.8.0/24 | 10.10.10.2 |
| DNS | Pi-hole (home, Docker on dockerpi) | 192.168.1.100 | — |
| Tunnel | WireGuard virtual network | 10.10.10.0/24 | — |

**DDNS:** `home.battistello.xyz` → `70.8.184.230` (Cloudflare, DNS Only)
**WireGuard Port:** UDP 51820

```
[Travel Devices 192.168.8.x]
        ↓
[Beryl AX — 192.168.8.1]
        ↓ WireGuard split tunnel
[UDR7 — 192.168.1.1]
        ↓
[Home Devices 192.168.1.x]
[Pi-hole — 192.168.1.100]
[TrueNAS — 192.168.1.4]
```

**Split tunnel:** Only `192.168.1.0/24` and `10.10.10.0/24` go through the VPN.
All other internet traffic exits the Beryl's local WAN directly.

## Setup — UDR7 WireGuard Server

1. UniFi Network → **Settings → VPN → VPN Server → Create New**
2. Type: WireGuard, Port: 51820
3. Check **Use Alternate Address for Clients** → `home.battistello.xyz`
4. Advanced → Gateway/Subnet → Manual: `10.10.10.10 / 24`
5. Add client **Travel Router**: Interface IP `10.10.10.2`, Remote Client Networks `192.168.8.0/24`
6. Download client config → edit before loading on Beryl (see below)
7. Verify firewall rule exists: **Settings → Firewall → Internet Local → Allow WireGuard Server (UDP)**

---

## Setup — Beryl WireGuard Client

Edit the downloaded `.conf` before loading:

```ini
[Interface]
PrivateKey = <auto-generated>
Address = 10.10.10.2/24
DNS = 192.168.1.100        # ← change from 10.10.10.10 to Pi-hole

[Peer]
PublicKey = <UDR7 public key>
Endpoint = home.battistello.xyz:51820
AllowedIPs = 192.168.1.0/24, 10.10.10.0/24   # ← change from 0.0.0.0/0
PersistentKeepalive = 25
```

Load: Beryl admin → **VPN → WireGuard Client → Add Manually** → name `Home-UDR7` → Enable.

### Beryl dnsmasq Upstream

```bash
ssh root@192.168.8.1
uci set dhcp.@dnsmasq[0].server='192.168.1.100'
uci commit dhcp
/etc/init.d/dnsmasq restart

# Verify
uci show dhcp.@dnsmasq[0].server
# dhcp.cfg01411c.server='192.168.1.100'
```

### Beryl Firewall (bidirectional LAN ↔ WireGuard)

> These do NOT survive a factory reset — reapply if needed.

```bash
ssh root@192.168.8.1

uci add firewall forwarding
uci set firewall.@forwarding[-1].src='wgclient1'
uci set firewall.@forwarding[-1].dest='lan'

uci add firewall forwarding
uci set firewall.@forwarding[-1].src='lan'
uci set firewall.@forwarding[-1].dest='wgclient1'

cat > /etc/firewall.user << 'EOF'
iptables -I INPUT -i wgclient1 -j ACCEPT
iptables -I FORWARD -i wgclient1 -o br-lan -j ACCEPT
iptables -I FORWARD -i br-lan -o wgclient1 -j ACCEPT
EOF

uci set firewall.wglan=include
uci set firewall.wglan.path='/etc/firewall.user'
uci set firewall.wglan.reload='1'

uci commit firewall
/etc/init.d/firewall restart
```

## Setup — Linux Travel Clients (systemd-resolved)

One-time per machine so `.home` queries route through the tunnel:

```bash
sudo mkdir -p /etc/systemd/resolved.conf.d/
sudo tee /etc/systemd/resolved.conf.d/home-domain.conf << 'EOF'
[Resolve]
Domains=~home
EOF
sudo systemctl restart systemd-resolved

# Verify
resolvectl query truenas.home   # should return 192.168.1.4
```

## Adding a New Travel Device

1. Connect to Beryl WiFi — gets DHCP from `192.168.8.0/24`
2. Find IP: `ssh root@192.168.8.1 'cat /tmp/dhcp.leases'` or check Beryl → Clients
3. Set static lease: Beryl admin → Clients → device → IP Binding
4. Add Pi-hole entry: `devicename.travel → 192.168.8.x`
5. If Linux: apply systemd-resolved drop-in (see above)

## Troubleshooting

**Tunnel won't connect**
```bash
dig home.battistello.xyz @1.1.1.1   # must return 70.8.184.230
ssh root@192.168.8.1 'wg show'
# Check UDR7: Settings → Firewall → Internet Local → Allow WireGuard Server (UDP 51820)
```

**Can't reach home LAN from travel (192.168.1.x unreachable)**
```bash
# AllowedIPs must include 192.168.1.0/24 — NOT 0.0.0.0/0
# Beryl → VPN → WireGuard Client → edit profile
```

**Can't reach travel LAN from home (192.168.8.x unreachable)**
```bash
# UDR7: peer → Remote Client Networks must include 192.168.8.0/24
ssh root@192.168.8.1 'iptables -L zone_wgclient1_forward -n'
# should show zone_lan_dest_ACCEPT
```

**.home names don't resolve from travel**
```bash
cat /etc/systemd/resolved.conf.d/home-domain.conf   # Domains=~home
uci show dhcp.@dnsmasq[0].server                    # must be 192.168.1.100
nslookup truenas.home 192.168.1.100                 # test Pi-hole directly
sudo resolvectl flush-caches && ping truenas.home
```

**.travel names don't resolve from home**
```bash
dig @192.168.1.100 survive.travel   # must return 192.168.8.2
# Check Pi-hole → Local DNS → DNS Records
```

**Tunnel works traveling but not at home**
- Hairpin NAT on UDR7 is automatic — wait 60–90s after plugging in
- Verify DDNS: `dig home.battistello.xyz @1.1.1.1`

**Beryl SSH is slow from home**
Connect a device directly to Beryl WiFi and SSH from there — avoids tunnel round-trip.
